# 🧬 AI Pathfinder 进化宪法

> 本文件定义项目自进化的边界、原则和质量红线。  
> 任何 AI Agent 在对项目做改动前，必须先读此文件并严格遵守。  
> 产品愿景和设计原则见 [VISION.md](VISION.md)。

---

## 不可变原则

1. **开源免费** — 项目必须保持 MIT 协议开源，不引入付费墙
2. **数据主权** — 用户数据仅存本地/session，不上传任何服务器
3. **资源只增不删** — 已有资源条目不可删除，只可标记 `deprecated: true`
4. **向后兼容** — LLM 路径输出的 JSON 格式（`weeks/resources/builders`）不可破坏现有结构
5. **最小依赖** — 不增加新的必选依赖（`requirements.txt`），可选功能放 `pyproject.toml` 的 `[project.optional-dependencies]`
6. **测试先行** — 任何功能变更必须有对应测试，现有测试不可被删除或跳过

---

## 变更安全分级

### 🟢 Safe — AI 可自主执行

- 修复 typo、语法错误
- 优化 CSS 样式和 UI 微调
- 补充/修正 i18n 翻译
- 增加测试用例
- 更新文档（README、CHANGELOG、注释）
- 代码格式化（遵循项目 lint 规则）
- 修复已有测试发现的 bug

### 🟡 Moderate — 需创建 PR 由人工 review

- 新增功能或页面
- 重构函数/模块（不改外部行为）
- 更新 `resources.yaml` 中的资源条目
- 修改 UI 布局或交互流程
- 升级依赖版本
- 性能优化

### 🔴 Dangerous — 必须人工决策

- 修改 System Prompt（`SYSTEM_PROMPT` / `CHAT_SYSTEM_PROMPT`）
- 修改 LLM 调用参数（temperature、max_tokens 等）
- 改变架构（文件结构、模块拆分）
- 修改数据格式（profile 编码、导出 JSON 结构）
- 修改进化规则本文件（`EVOLUTION.md`）
- 修改安全相关逻辑（API Key 处理、输入校验）

---

## 优先级排序原则

1. **安全漏洞** > 一切（立即修复）
2. **用户反馈中的 Bug** > 其他改进
3. **高频用户反馈** > AI 自主发现的优化
4. **用户体验改进** > 代码内部重构
5. **简单有效** > 复杂精巧

---

## 质量红线

- [ ] 所有现有测试必须通过（`pytest tests/`）
- [ ] 不引入新的安全漏洞（OWASP Top 10）
- [ ] 无 lint 报错（项目配置的 linter）
- [ ] 变更必须记录到 `CHANGELOG.md`
- [ ] 单次变更聚焦单一目标，不做"顺便改改"

## 进化灵感来源

AI 可主动从以下渠道发现优化机会：

1. **GitHub Issues**（label: `feedback` / `improvement` / `bug`）
2. **代码质量扫描** — lint 警告、类型检查、测试覆盖率
3. **内容质量审计** — 运行 `python scripts/audit_content.py`，修复内容问题
4. **优秀开源项目参考** — 学习同类项目的 UX、架构、最佳实践
5. **依赖更新** — 上游库的新版本带来的改进机会
6. **性能分析** — 页面加载速度、LLM 调用效率
7. **无障碍/国际化** — 多语言支持、可访问性改进

---

## 内容质量红线

资源内容是本项目的核心价值。以下规则不可违反：

1. **资源只增不删** — 已有资源标记 `deprecated: true` 而非删除
2. **新增必审** — 新增资源必须通过审计脚本的所有 HIGH 检查
3. **描述真实** — 描述必须准确反映资源内容，不夸大不缩水
4. **定期验证** — 每轮进化必须运行内容审计，修复发现的问题
5. **覆盖均衡** — 关注初学者和高级内容缺口，持续补充

详见 `CONTENT_QUALITY.md`。
