# 🧭 AI Pathfinder — 产品愿景

> **在 AI 时代学习 AI 不应该这么难。**

---

## 我们的信念

AI 正在重塑每一个行业。无数人想学习 AI——软件工程师想转型、传统算法工程师想升级、AI 从业者想跟上前沿——但他们面对的第一个障碍不是技术本身，而是**不知道从哪里开始**。

Coursera 上有 2000+ 门 AI 课程，YouTube 上有无穷无尽的教程，GitHub 上每天涌现新的开源项目。资源不是太少，而是太多。真正稀缺的不是内容，而是**一条刚好适合你的路径**。

AI Pathfinder 的信念是：**学习 AI 的门槛不应该是"找到正确的下一步"，而应该是学习本身。**

---

## 我们解决什么问题

**核心痛点：资源太多，不知道从哪开始。**

一个想从软件测试转向 AI 的工程师，和一个想读 AI 论文准备读研的学生，需要完全不同的学习路径。但大多数学习平台提供的是同一套路线图。

AI Pathfinder 做的事情很简单：
1. 了解你是谁（水平、目标、时间、方向、偏好）
2. 从人类策展的精选资源库中，用 AI 编排一条**刚好适合你**的路径
3. 帮你持续跟踪 AI 前沿，不在学完课程后迷失

## 设计原则

### 1. 人类策展 + AI 编排

资源由人类精心挑选（质量优先于数量），路径由 AI 根据个人情况编排。这种组合既保证了内容质量，又实现了千人千面的个性化。

### 2. 零摩擦启动

不知道该填什么？一键预设。不想配 API Key？侧边栏直接设置。第一次用 3 分钟内就能拿到一份有用的学习路径。

### 3. 数据主权

用户数据（画像、路径、进度）仅存本地 session。不上传、不追踪、不二次利用。学习是私人的事。

### 4. 供应商中立

支持 9+ 家 LLM 供应商，任何兼容 OpenAI 格式的 API 都能用。不绑定任何商业公司。

### 5. 开源透明

MIT 协议。所有推荐逻辑（Prompt、筛选规则、评审标准）完全公开。用户知道自己为什么被推荐了这些资源。

### 6. 内容即产品

资源库的质量直接决定产品价值。六维度内容质量管理（完整性、新鲜度、准确性、覆盖度、去重、描述质量）是产品的生命线。

## 架构哲学

### 当前：单体 Streamlit 应用（适合现阶段）

```
用户 → Streamlit UI → LLM API → 路径/对话
                ↕
         resources.yaml（人类策展的资源库）
```

**为什么是 Streamlit？**
- Python 生态，与 AI 工具链天然兼容
- 单文件部署，零前后端分离成本
- 适合 MVP 和快速迭代
- 社区友好，降低贡献门槛

**当前架构的边界**（到了这些信号就该考虑演进）：
- 并发用户 > 50 → 考虑前后端分离
- 资源 > 500 → 考虑数据库替代 YAML
- 需要用户账号体系 → 需要后端服务
- 需要多人协作标注 → 需要 CMS

### 文件职责划分原则

| 层 | 文件 | 变更频率 | 职责边界 |
|---|---|---|---|
| **入口** | app.py | 低 | 路由 + 初始化，不含业务逻辑 |
| **配置** | config.py | 中 | UI 常量 + LLM Prompt（未来可拆分） |
| **服务** | llm.py | 低 | LLM 交互，不碰 UI |
| **工具** | utils.py | 低 | 纯函数，无副作用 |
| **视图** | views/*.py | 高 | 每个页面独立，可单独理解和测试 |
| **数据** | resources.yaml | 中 | 唯一的资源真实来源 |
| **翻译** | i18n.py | 中 | 所有用户可见文本 |
